# Pourō-GPT Re Project Brief

## Product

Pourō-GPT Re is a quiet hand-drip brew guide, journal, and controlled-comparison workspace. It assumes responsibility for procedure, timing, water amount, and record structure so the user can focus on the coffee.

Core loop:

`Prepare → Brew → Reflect → Improve`

LAB loop:

`Baseline → change one variable → Candidate → Observe → Conclude → Next hypothesis`

## North Star

- Editorial Brew Journal
- Precision Instrument
- Contemporary Japanese Craft
- Graphite LAB Console

## Product Principles

- Quiet during extraction
- Precise without pretending certainty
- Recipe Truth with source metadata
- Local-first and No Hostage Data
- Human-editable implementation
- Accessible, responsive, and evidence-tested
- PWA first; native-ready later

## Primary Experience

`Home → Setup → Preview → Active Brew → Finish → Home Toast → History Detail → Brew Again`

## Core Methods

- 4:6
- New Hybrid
- 10 Pour
- Ice Brew

Additional methods remain governed by `03_RECIPE_TRUTH_V1.md`.

## Positioning

Main competitive reference: Filtru. This is a strategic reference, not a template to copy.

Pourō differentiates through:

- quiet and interruption-free brewing
- explicit Brew Runner states
- one-variable LAB Compare
- Recipe Truth and provenance
- user-owned exportable data
- restrained Japanese craft visual language

## Platform

- First: PWA, portrait-priority
- Future: iOS-native bridge/capabilities
- Hardware integrations: out of MVP; official SDK, public API, or licensed specifications only

## Success Definition

A release candidate is not complete until core flows work, recipe truth is preserved, data export/import is safe, required viewport checks pass, and unperformed physical-device or accessibility checks remain marked `NOT_RUN`.
