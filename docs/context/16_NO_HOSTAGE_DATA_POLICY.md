# No Hostage Data Policy

## Principle

Pourō must never hold user-created coffee data hostage.

This is a core competitive and ethical principle.

## Always Accessible

The user must retain basic access to:

- brew history
- saved recipes
- My Recipes
- LAB Compare records
- tasting notes
- JSON export
- JSON restore/import
- saved brew snapshots
- saved LAB snapshots

## Monetization Boundary

Paid features may provide:

- ad removal
- cloud sync
- high-resolution share cards
- advanced analysis
- LAB Studio future
- additional themes
- AI-assisted summaries future
- multi-device convenience

Paid features must not block:

- viewing local history
- viewing local recipes
- exporting user data as JSON
- restoring from JSON
- using accessibility features
- completing an active brew
- saving a completed brew

## Downgrade Rule

If a user loses premium status, existing user-created data must remain readable and exportable.

Premium-only rendering or cloud sync may stop, but the raw data must remain available.

## Ads

Ads must not appear:

- during Active Brew
- immediately before brew start
- before saving a brew
- during LAB evaluation
- before saving LAB results
- during import/export
- during error recovery

If ads are introduced, they must be non-interruptive and removable by purchase.

## Trust Statement

Pourō is a tool.

A trustworthy tool does not block the user from their own records.
