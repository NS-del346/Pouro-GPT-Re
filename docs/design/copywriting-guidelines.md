# Copywriting Guidelines

## Balance

- practical: 70%
- reassurance: 20%
- brand: 10%

## Persistent UI

Use direct, operational language. Short English state labels are acceptable:

`POUR`, `WAIT`, `NEXT`, `TOTAL`, `STEP`, `DONE`, `PAUSED`, `DRAWDOWN`, `TOOLS`, `SETTINGS`.

Do not replace canonical 4:6 terms:

- ベーシック
- より甘く
- より明るく
- 薄く
- より濃く
- さらに濃く

## Brand Copy

Poetic language belongs mainly in onboarding, About, App Store description, and first-run brand introduction. Do not place it on every screen.

If variant copy is used, select one of 3–5 screen-specific lines at session start and keep it fixed for that session. Do not randomize on every render.

## Brew Advice

Use:

`Observation → condition → optional experiment`

Example:

> 目標より40秒長くなりました。流れが遅く感じた場合は、次回は少し粗くする方法があります。

Do not state a diagnosis or certainty without sufficient data and research. No data means no advice.

## Empty States

Prefer practical reassurance:

> まだ記録はありません。最初の一杯を淹れると、ここに条件が残ります。

Avoid persistent sentimental statements that obscure function.
