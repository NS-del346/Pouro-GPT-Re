# Figma Workflow

## Sequence

1. Read authority context.
2. Create requirements checklist.
3. Build 2–3 major variants at 393×852.
4. Label each `EXPLORATION`.
5. Compare usability, accessibility, implementation cost, and authority compliance.
6. Mark one `RECOMMENDED`.
7. Obtain owner selection.
8. Mark selected direction `SELECTED`.
9. Expand to 375×667 and 430×932.
10. Define interaction/motion and Reduced Motion.
11. Implement in code.
12. Compare browser/device screenshots.
13. Mark verified design `IMPLEMENTED`.
14. Archive rejected variants.

## Boards

- Motion Foundation
- Home Redesign
- Active Brew State Model
- Preview Clarity
- Finish / Toast / History Detail
- Tools / Settings Separation

## Evidence

Each selected frame should record:

- viewport
- state
- source context references
- open questions
- implementation status
- last reviewed date

## Approval

Final Figma re-LOCK or library publish requires explicit owner approval. App Icon work remains deferred.
